Описание системы:
В данном проекте 2 микросервиса. Sensor и Server
Sensor:
1 end Point - регистрация сенсора на сервере. Он регигистирует сенсор, по имени, если такого имнени не регистрировалось,
то начинается автоматическая отправка сообщений на сервер. Ip сервера задается в application.yml
Swagger: {$ip:port}/swagger-ui/index.html

Server:
{$ip:port}/swagger-ui/index.html
Ip сервера задается в application.yml
в application.yml находится config, который задает время в минутах, за которое необходимо искать сенсоры,
колличество выводимых активных сенсоров, время за которое сенсоры переходят в неактивное состояние,
максимальная температура, минимальная температура

Для запуска приложения необходимо запустить compose.yml, который является файлом docker-compase, затем запустить
микросервис server, затем микросервис sensor

Не написаны тесты на приложение.
Затраченое время около 20 часов (3 проектирование, 11 написание кода и исправление багов, 3 тестирование, 3 документирование)
